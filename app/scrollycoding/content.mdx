## It's All About the Graph

To build a correct mental model for Web Audio API, we have to understand two concepts: **audio node** and **audio context**.

Audio nodes are the fundamental elements in the web audio world. **Audio always flows in a channel from the source node to destination nodes; along the way it may meet other nodes that affect your audio.** For example, `GainNode` can change the volume; `AnalyserNode` can extract frequency and waveform data from audio. (Just like the flow of life, we encounter different people and are consciously or unconsciously affected by them.) All the audio nodes and linkages between them consist of an **audio routing graph**. Audio context prepares all the tools we need, with which you can take your pen and draw your audio routing graph.

## !!steps Create the AudioContext

First, we need to create a context where we can say we've made the full preparation to operate the audio:

![AudioContext creation](/visualize-your-audio/audio-context-creation.png)

```js ! audio.js
const audioContext = new AudioContext()
```

## !!steps Create the Source Node

Then, a source node is necessary. Here the `audioElement` is an instance of `HTMLAudioElement`, which has the source URL of the audio. No worries if you don't understand—we'll delve into it later.

![Create media element source](/visualize-your-audio/create-media-element-source.png)

```js ! audio.js
const audioContext = new AudioContext()
const source = audioContext.createMediaElementSource(audioElement)
```

## !!steps Add the Destination

We need a destination node, which you don't need to create—instead, the audio context prepares it for you via `audioContext.destination`. Because audio always flows from a source node to destination nodes, we'll connect them next.

![Source and audioContext.destination nodes](/visualize-your-audio/source-and-destination-nodes.png)

```js ! audio.js
const audioContext = new AudioContext()
const source = audioContext.createMediaElementSource(audioElement)
// audioContext.destination is built-in
```

## !!steps Connect Source to Destination

Let's connect source with destination:

![Source connected to destination](/visualize-your-audio/source-connect-destination.png)

```js ! audio.js
const audioContext = new AudioContext()
const source = audioContext.createMediaElementSource(audioElement)

source.connect(audioContext.destination)
```

## !!steps Add the AnalyserNode

But nothing happens if we do only that. To extract visualization data from the audio, we need to add an `AnalyserNode` between the source node and destination node:

![Analyser in the audio chain](/visualize-your-audio/analyser-in-audio-chain.png)

```js ! audio.js
const audioContext = new AudioContext()
const source = audioContext.createMediaElementSource(audioElement)
const analyser = audioContext.createAnalyser()

source.connect(analyser).connect(audioContext.destination)
```

## !!steps Get the Frequency Data

Now that we have a good understanding of the Web Audio APIs, it's time to get the data we need for visualization.

Think about the animation first—it's just stacked frames. We need data for each frame; then the data changes for the next frame. For a frame, we seem to need an array, right? The size of the array is the x-axis and every element is probably the height—how high the symbol can reach.

`analyser.getByteFrequencyData()` is exactly what we need. It changes as the audio plays, and every time we call it, it gives us a snapshot of the frequency sample at that point. Just as we imagined, it's an array—perfect! The index of the array is frequency; the value of each element indicates how much power there is for that frequency.

> Why frequency? There's another method called `getByteTimeDomainData`—why don't we need that? I'm not going to jump into that rabbithole, because there's an amazing [viz post](https://pudding.cool/2018/02/waveforms/) about it. Go and read it!

```js ! audio.js
const audioContext = new AudioContext()
const source = audioContext.createMediaElementSource(audioElement)
const analyser = audioContext.createAnalyser()
source.connect(analyser).connect(audioContext.destination)

// Get frequency data for visualization
const data = new Uint8Array(analyser.frequencyBinCount)
analyser.getByteFrequencyData(data)
```
